# 豆花语音输入：基于豆包APP的语音输入助手

## 项目简介
本项目是一个基于 macOS 平台开发的豆包语音输入辅助工具。旨在通过快捷键托管和剪贴板保护机制，将豆包原生的“唤醒-说话-手动插入”流程，优化为类似 Typeless 的“按住说话、松开自动写入”体验。

## 核心能力
1. **全局快捷键托管与防误触**：监听用户按住/松开指定热键（默认右 Command）。内置长按阈值判定（默认 250ms），有效防止与系统或其它 App 的短按组合快捷键（如 `Cmd+C`）冲突。
2. **时序控制与自动插入**：长按触发豆包录音（`Option+D`），松开后自动发送 `Enter` 提交结果，并根据剪贴板状态智能判定是否需要主动执行粘贴（兜底机制）。
3. **空语音静默取消**：如果用户长按呼出语音但未说话，系统在超时未检测到剪贴板更新后，会自动发送 `Esc` 键关闭豆包悬浮窗，不产生任何多余的字符或粘贴动作。
4. **剪贴板保护**：在触发前拍摄剪贴板快照，等待豆包结果写入/或者取消录音后，无感恢复用户的原始剪贴板内容。
5. **可视化控制台**：提供本地 Web 界面（深色玻璃态风格），支持一键启停、参数热更新和实时运行日志查看。

## 技术架构

### 核心栈
- **语言**：Python 3
- **热键监听**：`pynput`
- **系统交互**：`subprocess` 调用 `pbpaste`/`pbcopy` 进行剪贴板读写
- **Web 服务**：内置 `http.server`，提供极轻量级控制面板

### 模块结构
- `app.py`：应用入口，初始化各组件并管理生命周期。
- `config.py`：配置管理，支持本地 JSON 文件读写。
- `hotkey_listener.py`：按键监听器，处理按压与释放事件。
- `keyboard_actions.py`：模拟键盘输出，注入组合键或单键。
- `clipboard_guard.py`：剪贴板快照与恢复。
- `orchestrator.py`：核心业务逻辑，统筹状态机与各组件的时序调用。
- `web_console.py`：本地 Web 控制台服务。
- `logging_buffer.py`：内存日志管理，供 Web 端轮询。

## 关键参数说明 (可在 Web 界面实时调节)
- `hold_threshold_ms`: 长按阈值（毫秒）。按住按键超过此时间才会唤醒豆包，低于此时间松开会被判定为普通按键/组合键，不予拦截。
- `normal_delay_ms`: 松开热键后，等待多少毫秒再发送 Enter 提交（给豆包识别一点缓冲时间）。
- `submit_settle_timeout_ms`: 提交 Enter 后，等待剪贴板内容发生变化的超时时间。若超时未变化，则判定为未说话或豆包异常，执行静默取消或兜底重试。
- `post_submit_grace_ms`: 剪贴板内容变化后，再等待多少毫秒以确保结果已原生粘贴进目标输入框（若豆包未原生粘贴，则由程序主动注入 `Cmd+V`）。
- `clipboard_restore_delay_ms`: 最终恢复原剪贴板前的小幅延迟缓冲。

## 开发与运行

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 运行单元测试
python3 -m unittest discover -s tests -p 'test_*.py'

# 3. 启动服务
python3 main.py
```
启动后访问 `http://127.0.0.1:8899` 即可打开控制台。

## 已知问题与优化方向
- 目前剪贴板仅使用文本方案 (`pbpaste/pbcopy`) 兜底，未来可升级为使用 `NSPasteboard` 以支持多类型数据（图片、RTF 等）。
- 可考虑使用 Swift/SwiftUI 重写为原生 macOS 应用，以获得更低的延迟和更好的系统权限兼容性。
